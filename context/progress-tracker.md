## Architecture Decisions

- Spec-generation task access is derived from the authenticated user plus `roomId`; client-supplied `projectId` is ignored at the API boundary.
- Generated specs are persisted to private Vercel Blob and surfaced through authenticated app download routes; Trigger task output returns the internal download path instead of a raw blob URL.
- Project spec metadata and preview content are exposed only through authenticated project-scoped app routes; the client never reads private Blob URLs directly.
